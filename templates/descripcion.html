Segmentación de Productos Amazon con K-Means

Agrupamiento no supervisado sobre el catálogo de Amazon India.

Planteamiento del problema

Plataformas de comercio electrónico como Amazon generan grandes volúmenes de transacciones con productos de distintos precios, categorías, valoraciones y popularidad. Esta información no tiene gran aprovechamiento si no es procesada por métodos de minería de datos capaces de encontrar patrones ocultos utiles para formar estrategias de precios, campañas de descuento y recomendaciones. Por ende: ¿es posible identificar grupos naturales o segmentos de mercado que compartan características similares de precio, descuento y popularidad?

Este proyecto aplica el algoritmo K-Means para segmentar automáticamente los productos del dataset de Amazon en grupos que permitan identificar patrones comerciales y apoyar decisiones de marketing a través de una aplicación web desarrollada en Flask.

Objetivos

  • Preparar y limpiar el dataset de Amazon Sales para su análisis.
  • Seleccionar las variables numéricas más representativas de cada producto.
  • Justificar el número óptimo de clústeres mediante el método del codo.
  • Aplicar K-Means con y visualizar los segmentos resultantes.
  • Extraer conclusiones utiles de negocio a partir del agrupamiento.

Metodología

Carga y limpieza de datos. Eliminación de símbolos, caracteres especiales y valores nulos.
Selección de variables. Elección de variables numéricas relevantes para realizar el clustering.
Normalización. Estandarización con StandardScaler para que todas las variables tengan una escala similar.
Método del codo. Ejecución de K-Means para K de 1 a 10, evaluando la inercia para seleccionar el K óptimo.
Aplicación de K-Means. Entrenamiento del modelo y asignación de los productos a los clústeres.
Reducción dimensional. Proyección de los datos en dos dimensiones con PCA para una mejor visualización.
Interpretación. Análisis del perfil de cada clúster con base en los centroides y estadísticas.
